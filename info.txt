Порядок запуску:

1) Розпакуйте архів або клонуйте репозиторій: git clone https://github.com/Balance4490/SystemActivityMonitorPrj.
2) Відкрийте файл рішення SystemActivityMonitor.sln у Visual Studio.
3) Запустіть проєкт кнопкою Start або через термінал: перейдіть у папку SystemActivityMonitor.UI та введіть dotnet run.
4)База даних: Проєкт налаштовано на використання SQLite. Встановлення SQL Server не потрібне. При першому запуску система автоматично створить файл system_monitor.db у папці виконання та ініціалізує необхідні таблиці.

Сценарій тестування:

Вхід: У вікні логіна введіть дані існуючого користувача або натисніть Register, щоб створити нового.
Робочий стіл: Після входу ви побачите емуляцію робочого столу. Запустіть Google Chrome або Visual Studio подвійним кліком.
Генерація навантаження:
У вікні Chrome натискайте кнопку + (New Tab). Кожна вкладка збільшує споживання RAM.
Закривайте вкладки x, щоб зменшити навантаження.
Моніторинг: Відкрийте вікно моніторингу. Спостерігайте за графіками CPU/RAM у реальному часі.
Керування (Admin): Спробуйте натиснути "Прибрати задачу" для запущеного Chrome — вікно закриється. Спробуйте видалити процес System — отримаєте відмову (захист ядра).
Аналітика: Натисніть кнопку "Аналіз ресурсів (Visitor)", щоб побачити згенерований текстовий звіт про розподіл пам'яті.
